UUID (Universally Unique Identifier) — это 128-битное число, используемое для идентификации информации. 
UUID гарантирует уникальность идентификатора без необходимости обращения к штуке, 
которая регистрирует id. 
Вероятность создания двух одинаковых UUID крайне мала, что делает их идеальными для идентификации объектов.

Для атомарности:
synchronized - блокировка на уровне методов или объектов

ReentrantLock - более гибкие блокировки (есть в коде)

AtomicInteger, AtomicReference - атомарные операции без блокировок

volatile - гарантирует видимость изменений между потоками

Для асинхронности:
ExecutorService - пулы потоков для параллельной обработки

CompletableFuture - асинхронные операции с цепочками вызовов

ConcurrentHashMap - потокобезопасные коллекции (есть у меня в коде)

BlockingQueue - очереди для обмена данными между потоками (тоже есть в коде)
