Зачем нужны Java и Docker?
1--Кроссплатформенность ("Write once, run anywhere" - WORA): Код компилируется в байт-код, который выполняется на Java Virtual Machine (JVM). Это значит, что одно и
 то же приложение может работать на Windows, Linux, macOS,
 не требуя переписывания. Это преимущество для корпоративного ПО.
2--Управление памятью: Автоматическая сборка мусора избавляет 
разработчика от многих рутинных и ошибкоопасных задач.

Docker: зачем нужны контейнеры?
1-- Контейнеры используют ядро хоста, поэтому они намного легче и быстрее запускаются, чем виртуальные машины. Образ можно
 перенести из облака на локальный сервер быстро.
2-- Проблема того, что у этого работает, у другого нет исчезает. Вы упаковываете приложение
 со всеми его зависимостями (библиотеки, версия Java, системные настройки) в единый образ. Этот образ
 запускается одинаково на любой системе, где есть Docker.
